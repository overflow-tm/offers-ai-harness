# offers-profile-review

Offers プロフィールをレビューし、強み・弱み・改善提案を提示するスキル。
ユーザーが改善を承認した場合のみプロフィールを更新し、変更前後の差分を表示する。

## プライバシーに関する注意

スキル実行開始時に、以下の注意事項をユーザーに伝えること。

> このスキルはプロフィール情報を Offers MCP サーバーから取得してレビューします。Offers 側ではセッション終了後にデータを保持しません。なお、AI エージェント側でのデータの取り扱いは、ご利用中のツールのプライバシーポリシーに従います。

## 実行フロー

### ステップ 1: プロフィール取得

MCP ツール `get_profile` を呼び出し、現在のプロフィール情報を取得する。

エラーが発生した場合は以下のメッセージを表示して終了する。

> プロフィールの取得に失敗しました。しばらく時間をおいて再度お試しください。

### ステップ 2: プロフィールレビュー

取得したプロフィールを分析し、以下の3項目を日本語で提示する。

#### レビュー観点

以下の観点でプロフィールを評価する。

- **完成度**: 各セクションが埋まっているか（スキル、職歴、アピールポイント、将来の意志など）
- **具体性**: アピールポイントや将来の意志が具体的に書かれているか
- **一貫性**: 職種・スキル・職歴・希望条件に矛盾がないか
- **訴求力**: 企業側から見て魅力的なプロフィールになっているか
- **条件設定**: 希望年収・転職意欲・勤務形態が適切に設定されているか

#### 出力フォーマット

```
## プロフィールレビュー結果

### 強み
- (プロフィールから読み取れるポジティブな点を箇条書き)

### 改善が望ましい点
- (不足している情報や改善できる点を箇条書き)

### 改善提案
以下の項目を更新することで、プロフィールの訴求力を高められます。

| # | 項目 | 現在の値 | 提案する値 | 理由 |
|---|------|----------|-----------|------|
| 1 | appealPoint | (現在値 or 未設定) | (提案値) | (なぜこの変更が有効か) |
| 2 | ... | ... | ... | ... |

※ 以下の項目は MCP 経由では更新できないため、Offers Web で直接編集してください。
- (該当する項目があれば列挙: スキル、職歴、氏名、メールアドレス)
```

#### 特殊ケースの対応

- **スキルが 0 件の場合**: レビューは実施するが、改善が望ましい点に「スキルを追加することで企業からの検索にヒットしやすくなります。Offers Web でスキルを追加してください。」と記載する。
- **アピールポイントや将来の意志が未設定の場合**: 職種・スキル・職歴から推測した文案を提案値として提示する。
- **改善提案が無い場合**: 「現在のプロフィールは十分に充実しています。」と伝え、ステップ 3 には進まない。

### ステップ 3: ユーザー承認の確認

改善提案を提示した後、以下のように承認を求める。

> 上記の改善提案を適用しますか？
> - 全て適用する場合: 「はい」「適用して」などと回答してください
> - 一部のみ適用する場合: 適用したい提案の番号を指定してください（例: 「1と3を適用」）
> - 適用しない場合: 「いいえ」「やめておく」などと回答してください

**重要**: ユーザーが明示的に承認するまで `update_profile` を呼び出してはならない。
ユーザーが拒否した場合は「承知しました。プロフィールは変更しません。」と伝えて終了する。

### ステップ 4: プロフィール更新

ユーザーが承認した提案のみを `update_profile` で適用する。

#### update_profile で更新可能なフィールド

| フィールド名 | 型 | 有効な値 |
|-------------|---|---------|
| occupation | enum | ENGINEER, DESIGNER, DATA_ANALYST, PM |
| careerMotivation | enum | NOW, AFTER_3_MONTHS, AFTER_6_MONTHS, AFTER_1_YEAR, POSITIVE, NONE |
| motivation | enum | HIGH, MEDIUM, LOW |
| expectedAnnualPay | number | 万円単位の数値 |
| currentAnnualPay | number | 万円単位の数値 |
| workingPlaceType | enum | office, remote, hybrid |
| appealPoint | string | 自由記述 |
| willInTheFuture | string | 自由記述 |
| educationalBackground | enum | HIGH_SCHOOL, VOCATIONAL_SCHOOL, JUNIOR_COLLEGE, UNIVERSITY, GRADUATE_SCHOOL, MASTER, DOCTOR, OTHER |
| intentionality | enum | TECHNOLOGY, ORGANIZATION, PRODUCT |
| pref | enum | TOKYO, OSAKA, その他都道府県 |

#### update_profile で追加で更新可能なフィールド

| フィールド名 | 型 | 説明 |
|-------------|---|------|
| subOccupations | array | サブ職種（`[{ subOccupationId: "ID", subOccupationExp: "年数" }]`） |

#### MCP 経由で更新できないフィールド

以下のフィールドは `update_profile` では変更できない。改善提案に含まれる場合は「Offers Web で直接編集してください」と案内する。

- skills（スキル）
- careers（職歴）
- name（氏名）
- email（メールアドレス）
- website_urls（Web サイト URL）
- company_preferences（希望企業の規模・業界）

エラーが発生した場合は以下のメッセージを表示する。

> プロフィールの更新に失敗しました。しばらく時間をおいて再度お試しください。変更は適用されていません。

### ステップ 5: 変更確認

更新成功後、`get_profile` を再度呼び出し、変更前後の差分を表示する。

#### 差分表示フォーマット

```
## 更新結果

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| appealPoint | (変更前の値) | (変更後の値) |
| ... | ... | ... |

プロフィールの更新が完了しました。
```

## MCP 呼び出し制約

合計 MCP 呼び出し回数は最大 3 回とする。

1. `get_profile` -- プロフィール取得（必須）
2. `update_profile` -- プロフィール更新（ユーザー承認時のみ）
3. `get_profile` -- 更新後の確認（更新実行時のみ）

ユーザーが更新を拒否した場合、MCP 呼び出しは 1 回で終了する。

## エラーハンドリング

- `get_profile` 失敗: レビューを実施せず、再試行を促すメッセージを表示して終了
- `update_profile` 失敗: 変更が適用されていない旨を伝え、再試行を促す
- 確認用 `get_profile` 失敗: 「更新は実行されましたが、確認の取得に失敗しました。Offers Web でプロフィールを確認してください。」と案内

## 言語

全ての出力は日本語で行う。
